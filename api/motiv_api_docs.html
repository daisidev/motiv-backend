Base Configuration

Base URL: /api/v1

Authentication: Bearer Token (JWT) - Include in headers as Authorization: Bearer <token>

Authentication Endpoints

User Signup

POST /api/v1/auth/signup

Request Body:

{
  "name": "string (required, min 2 chars)",
  "username": "string (required, 3-30 chars)",
  "email": "string (required, valid email)",
  "password": "string (required, min 6 chars)",
  "confirmPassword": "string (required)",
  "role": "guest" | "host" (optional, default: guest)
}

Response (201 - Success):

{
  "token": "string",
  "user": {
    "id": "uuid",
    "name": "string",
    "username": "string",
    "email": "string",
    "avatar": "string",
    "role": "guest" | "host" | "admin" | "superhost"
  }
}

User Login

POST /api/v1/auth/login

Request Body:

{
  "email": "string (required)",
  "password": "string (required)"
}

Google OAuth

POST /api/v1/auth/google

Request Body:

{
  "idToken": "string (required)"
}

Response:

{
  "token": "string",
  "user": { /* User object */ },
  "isNewUser": boolean
}

Password Reset

POST /api/v1/auth/forgot-password

Request Body:

{
  "email": "string (required)"
}
POST /api/v1/auth/reset-password

Request Body:

{
  "token": "string (required)",
  "newPassword": "string (required, min 6 chars)",
  "confirmPassword": "string (required)"
}

Event Endpoints

List Events with Filtering

GET /api/v1/events

Query Parameters:

Parameter Type Description Default
page integer Page number 1
limit integer Items per page (max: 100) 12
search string Search in title/description -
tags string Comma-separated tags -
location string Location filter -
event_type enum "free" or "ticketed" -
date_from date Filter from date (YYYY-MM-DD) -
date_to date Filter until date (YYYY-MM-DD) -

Important Implementation Notes

Authentication: Most endpoints require a Bearer token in the Authorization header.

Date Formats:

  • Dates: YYYY-MM-DD
  • Times: HH:MM
  • DateTime: ISO 8601 format

Additional Notes:

  • Pagination: List endpoints support page and limit parameters
  • File Uploads: Banner images should be uploaded separately and URL provided in bannerImageURL
  • Currency: All monetary values are in the base currency (likely Naira), with Paystack amounts in kobo
  • QR Codes: Generated automatically for tickets and used for check-in functionality
  • Webhooks: Payment webhooks are handled automatically by the backend
  • Rate Limiting: Be mindful of API rate limits, especially for search and analytics endpoints
